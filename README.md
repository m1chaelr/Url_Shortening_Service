# URL Shortening Service
This activity involves creating a RESTful API Url Shorting Service.

## Requirements
1. The API should allow users to create a shortened URL for a given long URL.
2. The API should allow users to retrieve the original long URL using the shortened URL.
3. The API should handle edge cases, such as invalid URLs and duplicate entries.
4. Update an existing short URL.
5. Delete an existing short URL.
6. Get Statistics on the short URL (e.g., number of times accessed).
